hexo_commands

发表于 2018-12-10 | 分类于 tool

init

在指定文件夹新建网站

new

使用layout模版新建一篇文章，layout默认为_config.yml中的default_layout

generate

生成静态文件

选项 含义
-d, –deploy 文件生成后立即部署网站
-w, –watch 监视文件变动

publish

发表草稿

server

启动服务器，默认网址: http://localhost:4000/

选项 含义
-p, –port 重设端口
-s, –static 只使用静态文件
-l, –log 启动日志记录，使用覆盖记录格式

deploy

部署网站

选项 含义
-p, –port 重设端口

render

渲染文件

选项 含义
-o, –output 设置输出路径

migrate

从其他博客系统迁移内容

clean

清除缓存文件(db.json)和已生成的静态文件(public)

在某些情况(尤其是更换主题后)，发现对站点的更改不生效，可能需要运行该命令

list

列表网站资料

version

查看hexo版本

安全模式

以安全模式下，不会载入插件和脚本

当安装新插件遭遇问题时，可以尝试以安全模式重新执行

调试模式

在终端中显示调试信息并记录到debug.log

简洁模式

隐藏终端信息

自定义配置文件的路径

自定义配置文件的路径，执行后将不再使用_config.yml

显示草稿

显示source/_drafts文件夹中的草稿文章

自定义CWD

自定义当前工作目录(Current working directory)的路径

js完美验证身份证号码

发表于 2018-12-08 | 分类于 前端

身份证号码：17位数字本体码 + 1位数字校验码

身份证号码规则

15位校验规则 6位地址编码 + 6位出生日期 + 3位顺序号

18位校验规则 6位地址编码 + 8位出生日期 + 3位顺序号 + 1位校验位

出生日期 (18|19|20)?\d{2}(0[1-9]|1[12])(0[1-9]|[12]\d|3[01])

身份证正则表达式：6位地址码 + (2位世纪码)? + 年月日 + 3位顺序码 + (1位校验码)?
/^\d{6}(18|19|20)?\d{2}(0[1-9]|1[12])(0[1-9]|[12]\d|3[01])\d{3}(\d|X)?$/i
